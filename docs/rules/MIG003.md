# MIG003 - Column change forces a full table rewrite

`AlterColumn` that changes the stored type (or flips a column to non-nullable)
rewrites every row while holding an exclusive lock. On a large table this is a
long outage.

## Bad

```csharp
migrationBuilder.AlterColumn<string>(
    name: "Status",
    table: "Orders",
    nullable: false);
```

## Prefer additive steps

1. Add a new nullable column with the target type.
2. Backfill it in batches, out of band from the migration.
3. Swap reads and writes over in application code.
4. Drop the old column later (see MIG001).

## Suppressing

Use the reviewed marker on tables small enough that the rewrite is instant.
