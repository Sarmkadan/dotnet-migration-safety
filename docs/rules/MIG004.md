# MIG004 - Migration drops a table

`DropTable` deletes the table and all of its rows. Like MIG001 it cannot be rolled
forward, and any code still referencing the table will fail immediately.

## Bad

```csharp
migrationBuilder.DropTable(name: "LegacyAudit");
```

## Prefer a staged retirement

1. Confirm nothing in the codebase references the table.
2. Rename it (or stop writing to it) for one release as a safety window.
3. Drop it in a dedicated migration that is easy to spot in review.

## Suppressing

```csharp
// migration-safety:reviewed LegacyAudit confirmed unused since v3.0
migrationBuilder.DropTable(name: "LegacyAudit");
```
