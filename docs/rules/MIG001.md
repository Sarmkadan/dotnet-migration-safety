# MIG001 - Migration drops a column

`DropColumn` deletes a column and every value stored in it. The operation is not
reversible by a roll-forward, and during a rolling deploy older application
instances may still be reading the column when the migration runs.

## Bad

```csharp
migrationBuilder.DropColumn(name: "Notes", table: "Orders");
```

## Prefer an expand / contract sequence

1. Ship application code that no longer reads or writes the column.
2. Wait until every running instance is on that version.
3. Drop the column in a later migration.

## Suppressing

If you have genuinely reviewed the drop, mark the statement so the analyzer stays
quiet and the decision is visible in the diff:

```csharp
// migration-safety:reviewed Notes was never populated in any environment
migrationBuilder.DropColumn(name: "Notes", table: "Orders");
```
