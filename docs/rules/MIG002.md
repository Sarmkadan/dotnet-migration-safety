# MIG002 - Index is created without the concurrent option

`CreateIndex` emits a plain `CREATE INDEX`, which holds a lock that blocks writes
to the table until the whole index is built. On a large table that can stall the
application for minutes.

## Bad

```csharp
migrationBuilder.CreateIndex(
    name: "IX_Orders_CustomerId",
    table: "Orders",
    column: "CustomerId");
```

## Prefer a concurrent build

On PostgreSQL, build the index outside the migration's transaction:

```csharp
migrationBuilder.Sql(
    "CREATE INDEX CONCURRENTLY IF NOT EXISTS \"IX_Orders_CustomerId\" ON \"Orders\" (\"CustomerId\");",
    suppressTransaction: true);
```

SQL Server users on Enterprise editions can pass `WITH (ONLINE = ON)`.

Small lookup tables are fine to index inline; use the reviewed marker to opt out.
