# Suppression Comment Grammar

## Overview

The `SuppressionComment.IsReviewed()` method uses a tolerant parser to detect the `migration-safety:reviewed` marker in leading comments. This parser is intentionally designed to survive code formatting tools (dotnet format, Rider), extra whitespace, comment style changes, or hand-edited comments.

## Accepted Grammar

The parser accepts the following grammar (case-insensitive, whitespace-tolerant):

```
comment = single_line_comment | multi_line_comment

single_line_comment = "//" <content>
multi_line_comment = "/*" <content> "*/"

content = marker_prefix marker_suffix?

marker_prefix = "migration" whitespace "-"? whitespace "safety" whitespace ":" whitespace "reviewed"

marker_suffix = "(" <any_text> ")" <any_text>? | <any_text>

whitespace = <one_or_more_whitespace_characters>
any_text = <any_character_sequence>
```

## Examples of Accepted Formats

### Standard Formats
- `// migration-safety:reviewed safe to drop`
- `/* migration-safety:reviewed safe to drop */`


### Whitespace Variations (tolerated)
- `// migration-safety : reviewed safe to drop` (space around colon)
- `// migration-safety: reviewed safe to drop` (space after colon)
- `//    migration-safety   :   reviewed   (MIG001)   safe to drop` (multiple spaces)
- `//\tmigration-safety:\treviewed safe to drop` (tabs)

### Case Variations (tolerated)
- `// MIGRATION-SAFETY:REVIEWED safe to drop` (uppercase)
- `// Migration-Safety:Reviewed safe to drop` (mixed case)
- `// mIgRaTiOn-sAfEtY:rEvIeWeD safe to drop` (random case)

### With Diagnostic IDs (tolerated)
- `// migration-safety:reviewed (MIG001) TODO: document why this is safe`
- `// migration-safety:reviewed (MIG001:DropColumn) safe operation`
- `// migration-safety:reviewed (MIG001:DropColumn) reviewed by Alice on 2024-01-15`

### Multi-line Comments (tolerated)
- `/* migration-safety:reviewed safe to drop */`
- `/* MIGRATION-SAFETY : REVIEWED (mig001) */`
- `/* migration-safety:reviewed (MIG001) TODO: document */`

### With Additional Text (tolerated)
- `// migration-safety:reviewed reviewed by Alice on 2024-01-15`
- `// migration-safety:reviewed safe to drop column Notes`
- `// migration-safety:reviewed (MIG001) reviewed by Bob, 2024-01-15`

## Examples of Rejected Formats

### Wrong Marker Text
- `// migration-safety:approved safe to drop` ❌ (must contain "reviewed")
- `// migration-safety:review safe to drop` ❌ (misspelled "reviewed")
- `// safety:reviewed` ❌ (missing "migration-")

### Not in Leading Trivia
- `migrationBuilder.DropColumn(...); // migration-safety:reviewed` ❌ (trailing comment, not leading)

### Missing Marker
- No comment at all ❌
- Comment without the marker ❌

## Implementation Details

The parser uses:
1. **Regex-based matching**: A case-insensitive regex pattern with whitespace tolerance
2. **Comment extraction**: Proper handling of both `//` and `/* */` comment syntax
3. **Leading trivia only**: Only checks comments in the leading trivia of statements

## Regex Pattern

```csharp
private static readonly Regex MarkerPattern = new Regex(
    @"migration\\s*-?\\s*safety\\s*:\\s*reviewed\\s*(\\([^)]*\\))?\\s*(.*)",
    RegexOptions.IgnoreCase | RegexOptions.Compiled);
```

This pattern matches:
- `migration` followed by optional whitespace, optional hyphen, optional whitespace
- `safety` followed by optional whitespace, colon, optional whitespace
- `reviewed` followed by optional whitespace
- Optional parentheses with any content
- Optional trailing text

## Round-Trip Guarantee

The parser is designed to handle comments generated by the `MarkReviewedCodeFixProvider` even after formatting perturbations:

```csharp
// migration-safety:reviewed (MIG001) TODO: document why this is safe
```

This format will be correctly parsed even if:
- dotnet format adds extra spaces: `//    migration-safety   :   reviewed   (MIG001)   TODO: document...`
- Rider reformats it: `/* MIGRATION-SAFETY : REVIEWED (mig001) TODO: document... */`
- A developer manually edits it with different casing or spacing

## Related Components

- **Writer**: `MarkReviewedCodeFixProvider` (generates comments in format: `// migration-safety:reviewed (MIG001) TODO: document why this is safe`)
- **Reader**: `SuppressionComment.IsReviewed()` (parses comments with tolerance for formatting drift)
- **Contract**: The marker text `migration-safety:reviewed` must be present in leading comments

## Migration Safety Analyzer

This tolerant parsing ensures that suppression comments survive:
- Code formatting tools (dotnet format, Rider, Visual Studio)
- Manual editing by developers
- Different coding styles and conventions
- Review processes where comments may be reformatted

The analyzer can then reliably detect reviewed operations even when the code has been reformatted.
